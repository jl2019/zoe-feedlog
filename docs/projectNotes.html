First, git commit subjects, should we create topic keywords.

Example commit message:

config: set ruby version:2.6.2
  • "config" declares which type of task this commit encapsulated. here we're updating project configurations in whichever files were commited"
  • "set ruby version" is a description of the action taken
  • "2.6.2" is the value for that config.

This is the simplest use case, and open to further development
